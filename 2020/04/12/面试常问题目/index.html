Java基础

  • 重载和重写的区别

重载发生在同一个类中，方法名相同，但参数类型、参数个数、参数顺序、方法返回值和访问修饰符可以不同。

重写发生在子类中，方法名、参数列表必须相同，返回值范围小于等于父类，抛出的异常小于等于父类。

  • String、StringBuffer和StringBuilder的区别是什么？

String是不可变字符串，所以是线程安全的，适合操作少量的数据。

StringBuffer对方法添加了同步锁，所以是线程安全的，适合多线程在字符串缓冲区下操作大量数据。

StringBuilder是非线程安全的，适合单线程在字符串缓冲区下操作大量数据。

  • 接口与抽象类的区别是什么？

接口的方法默认被public修饰，方法在接口中不能有实现，而抽象类可以有非抽象方法。

接口中除了static和final变量，不能有其他变量。

一个类可以实现多个接口，但只能继承一个抽象类。

设计层面来说，抽象是对类的抽象，是一种模板设计；而接口是对方法的抽象，是一种行为规范。

  • == 与 equals 的区别

== 判断两个对象是否是同一个对象（基本数据类型比较值，引用类型比较内存地址）

equals 有两种情况：一是该类没有覆盖equals方法，则等价于 == ；二是该类覆盖了equals方法，则判断他们的内容是否相等

  • hashCode 与 equals

若两个对象相等，则hashCode一定相等

若两个对象相等，对两个对象分别调用equals方法都返回true

两个对象有相同的hashCode，他们也不一定相等

equals方法被覆盖，则hashCode方法也必须被覆盖

Java集合

  • ArrayList和LinkedList的区别

是否线程安全：都是不同步的，即都不保证线程安全

底层数据结构：ArrayList使用的是Object数组，LinkedList使用的是双向链表

插入和删除的复杂度不同

是否支持快速随机访问：ArrayList实现了RandomAccess接口，而LinkedList没有实现

内存空间占用：ArrayList会有一定的未使用空间，而LinkedList占用空间大主要体现在它的存储结构上

Java多线程

  • 进程与线程的区别

进程是程序的一次执行过程，是系统运行程序的基本单位。线程是操作系统能够进行运算调度的最小单位。

一个进程可以产生多个线程，这些线程共享该进程的堆和方法区资源，但每个线程都有自己的程序计数器、虚拟机栈和本地方法栈。在线程之间切换负担要比进程小的多。

  • 什么是上下文切换？

简单概括来说就是当一个线程执行完一个CPU时间片后，切换到另一个线程前会保存自己的状态，以便轮到自己运行的时候可以再加载这个任务。任务从保存到再加载的过程就是一次上下文切换。

  • 什么是死锁？如何避免死锁？

死锁的四个条件：互斥条件、请求与保持条件、不剥夺条件、循环等待条件

  • volatile关键字

保持变量的内存可见性、防止指令重排

https://zhuanlan.zhihu.com/p/73126227

  • synchronized关键字

被synchronized修饰的方法或代码块，每次只有一个线程能够访问该方法或代码块，因此保证了线程同步。

使用方法：对象锁、类锁

原子性、可见性、有序性、可重入性。monitorenter、monitorexit

synchronized的四个状态：无锁、偏向锁、轻量级锁、重量级锁

每一个锁都有一个monitor对象与之关联，当一个monitor被某个线程持有后，它便处于锁定状态

  • synchronized和ReentrantLock区别

API层面：这两种方式最大区别就是对于Synchronized来说，它是java语言的关键字，是原生语法层面的互斥，需要jvm实现。而ReentrantLock它是JDK 1.5之后提供的API层面的互斥锁，需要lock()和unlock()方法配合try/finally语句块来完成。

等待可中断：等待可中断是指当持有锁的线程长期不释放锁的时候，正在等待的线程可以选择放弃等待，改为处理其他事情。synchronized不能被中断，而ReentrantLock可以被中断。

公平锁：公平锁是指多个线程在等待同一个锁时，必须按照申请的时间顺序来依次获得锁；而非公平锁则不能保证这一点。非公平锁在锁被释放时，任何一个等待锁的线程都有机会获得锁。

Java虚拟机

  • new 一个对象的过程

第一步，根据new指令的参数，先在常量池检查是否能找到该参数对应的引用，并且检查该引用是否已经进行了类加载、解析和初始化，若没有则先进行类加载过程。

第二步，为该对象分配内存区域，有两种分配方式，指针碰撞和空闲列表，为了解决同步问题，可以使用TLAB来处理，只有需要重新分配新的TLAB时，才加上同步锁。

第三步，虚拟机将分配到的内存空间都初始化零值，这样使得即使没有赋初始值也可以访问到该对象的零值。

第四步，对该对象进行必要的设置，如类的元信息、哈希码、GC分代年龄等，并将其储存到对象头中。

第五步，执行init方法，按照程序员的意愿进行初始化。这样一个对象就创建完成了。

  • 乐观锁和悲观锁

乐观锁：即每次拿数据的时候都不上锁，但是会在提交更新的时候判断一下该数据是否被修改。适用于读多写少的场景。

乐观锁有两种实现方式：

  1. 检查数据库版本。数据库维护一个version字段，数据每更新一次则version值加1。读取数据的时候会将version一起读出，提交更新时数据库会检查version是否相等，若相等则更新；若不相等则认为是过期数据，不更新（即提交之前已经有人提交过了，因此version值增加，则一并读取的version值会比更新后的version值小）
  2. 使用时间戳。基本和第一个方法一样，也是检查时间戳是否相等。

悲观锁：即每次拿数据的时候都会上锁，无论读写，这样别人想访问该数据就需要获得锁。

Redis

  • 为什么要用redis/为什么要用缓存

高性能：对于一些修改不太频繁的数据，可以扔缓存里，这样重新读取的速度比从数据库里读取要快

高并发：MySQL不适合高并发，当请求数量过多时MySQL负荷过大，但redis可以支持高并发，因为其就是简单的key-value模式

  • 为什么要用redis缓存而不是map缓存

缓存分为本地缓存和分布式缓存。使用Java自带的map实现的是本地缓存，主要特点是轻量以及快速，生命周期随着jvm的销毁而结束，并且在多实例的情况下，每个实例都需要各自保存一份缓存，不具有一致性。

  • redis和memcached的对比

redis单核，适合小数据，支持的数据类型较多，同时支持持久化操作，可以将内存中的数据保存到磁盘中，为了在重启时能够再次加载使用。

memcached多核，适合大数据，支持的数据类型较少，不支持数据持久化存储。

  • redis的线程模型？为什么redis单线程模型效率也这么高？

redis采用单线程模型，用IO多路复用机制同时监听多个socket，每次从队列中取出一个socket进行处理。redis是纯内存操作，核心是基于非阻塞的IO多路复用机制，避免了多线程繁琐的上下文切换问题。

  • redis常见数据结构以及使用场景

String：最简单的key-value模型，应用：记录微博数，粉丝数等。

Hash：一个string类型的key和value的映射表，适合用于储存对象，操作的时候可以仅仅修改对象中的某个字段。

List：list的实现为一个双向链表，支持反向查询和遍历，主要应用场景有粉丝列表、关注列表、消息列表等。

Set：set与list类似，不过会自动去重，同时set可以实现交集、并集、差集等操作，主要应用场景有共同关注、共同好友、共同喜好等等。

Sorted Set：该类型增加了一个权重参数score，使得集合中的元素能够按照score来进行排序，

  • redis设置过期时间

定期删除：redis默认每隔100ms就随机抽取一些设置了过期时间的key，检查其是否过期，若过期了则删除。

惰性删除：若有很多已过期但未被删除的数据，除非你的系统去排查，否则redis不会主动删除。

  • redis内存淘汰机制

volatile-lru：从已设置过期时间的数据集中挑选最近最少使用的数据淘汰。

volatile-ttl：从已设置过期时间的数据集中挑选将要过期的数据淘汰。

volatile-random：从已设置过期时间的数据集中任意选择数据淘汰。

allkeys-lru：当内存不足以容纳新数据时，移除最近最少使用的数据。

allkeys-random：从数据集中任意选择数据淘汰。

no-eviction：禁止驱逐数据，当内存不足时，新写入操作会报错。

设计模式

  • 简单工厂

简单工厂模式的实质是由一个工厂类根据传入的参数，动态决定应该创建哪一个产品类。

Spring中的BeanFactory就是简单工厂模式的体现，根据传入一个唯一的标识来获得Bean对象，但是否是在传入参数后创建还是传入参数前创建这个要根据具体情况来定。

  • 工厂方法

定义一个用于创建对象的接口，让子类决定实例化哪一个类。Factory Method使一个类的实例化延迟到其子类。

Spring中的FactoryBean就是典型的工厂方法模式。

  • 单例模式

保证一个类仅有一个实例，并提供一个访问它的全局访问点。

Spring中的单例模式完成了后半句话，即提供了全局的访问点BeanFactory。但没有从构造器级别去控制单例，这是因为Spring管理的是是任意的Java对象。

  • 适配器

将一个类的接口转换成客户希望的另外一个接口。Adapter模式使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。

  • 包装器

动态地给一个对象添加一些额外的职责。就增加功能来说，Decorator模式相比生成子类更为灵活。

  • 代理

为其他对象提供一种代理以控制对这个对象的访问。

  • 观察者

定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都得到通知并被自动更新。

  • 策略

定义一系列的算法，把它们一个个封装起来，并且使它们可相互替换。本模式使得算法可独立于使用它的客户而变化。

  • 模板方法

定义一个操作中的算法的骨架，而将一些步骤延迟到子类中。Template Method使得子类可以不改变一个算法的结构即可重定义该算法的某些特定步骤。
